# Руководство проекта (CLAUDE.md)

Этот файл содержит правила разработки, сборки, тестирования, а также стандарты ведения базы знаний проекта **ADC Input Model** в формате **LLM-Wiki**.

---

## 1. Команды для разработки и тестирования

В ходе разработки математического ядра (на Python 3) и веб-интерфейса (на WebAssembly/HTML/Plotly) используются следующие команды:

### Математическое ядро (Python)
- **Запуск тестов решателя**: `python -m unittest discover -s tests` (после создания папки `tests`).
- **Запуск отдельного скрипта верификации**: `python src/verify_model.py` (после реализации).
- **Сборка N-мерной сетки**: `python src/generate_sweep.py`.

### Веб-интерфейс (Wasm / Shinylive)
- **Локальный запуск Shinylive веб-сервера**: `shiny run --reload src/app` (или аналогичная команда для PyScript/HTTP-сервера: `python -m http.server 8000`).

## 2. Стандарты ведения базы знаний (LLM-Wiki)

База знаний проекта является основным источником правды для архитектурных решений, математических моделей и хода разработки. Проект разделен на три слоя:
1. `raw/` — первоисточники, ТЗ, статьи (неизменяемые).
2. `wiki/` — компилируемые Markdown-заметки о концепциях и сущностях.
3. `CLAUDE.md` — этот файл правил и руководств.

### Правила изменения Wiki
- **Импорт документов**: При появлении нового файла в `raw/`:
  1. Описать его в [index.md](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/wiki/index.md).
  2. Выделить ключевые концепции и создать новые `.md` заметки в `wiki/concepts/` или `wiki/entities/`.
  3. Добавить запись в [log.md](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/wiki/log.md).
- **Связывание страниц**: Каждая новая страница должна иметь обратные ссылки (backlinks) и быть связана с индексным файлом.
- **Оформление**: Использовать синтаксис Markdown и разметку формул LaTeX (`\(...\)` для инлайна и `\[...\]` для блоков).
- **Ссылки на файлы**: Все ссылки на файлы репозитория в общении и артефактах должны быть кликабельными с использованием схемы `file:///` и прямых слешей (например, `[TЗ.txt](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/raw/TЗ.txt)`). Внутри страниц Wiki допускаются относительные ссылки вида `[ТЗ](../raw/TЗ.txt)`.

## 3. Стандарты оформления кода

- **Математика**: Решение дифференциальных уравнений RC-цепи должно быть выполнено во временной области с использованием жестких решателей (`scipy.integrate.solve_ivp` с методом `Radau` или `BDF`).
- **Библиотеки**: Разрешено использовать только `numpy`, `scipy` и `plotly` (или `matplotlib` для локальных тестов).
- **Код**: Чистый Python 3 с типизацией (`typing`), подробным логированием и тестами.
