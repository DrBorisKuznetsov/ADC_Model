# Эффект DC bias в керамических конденсаторах (dc_bias.md)

**Эффект DC bias (зависимость от постоянного напряжения)** — это физическое свойство многослойных керамических конденсаторов (MLCC) на основе диэлектриков с высокой диэлектрической проницаемостью (класс II, такие как X7R, X5R, Y5V), при котором их реальная емкость существенно уменьшается при приложении постоянного напряжения смещения. 

Конденсаторы класса I (например, C0G/NP0) практически не подвержены этому эффекту и сохраняют стабильную емкость.

---

## 1. Моделирование нелинейной емкости $C(V)$

Характеристика зависимости емкости от приложенного напряжения $V$ является симметричной: $C(V) = C(-V)$. Для моделирования используются интерполяционные таблицы (Look-Up Tables, LUT), где значения напряжения нормированы относительно номинального напряжения конденсатора $V_{rated}$.

### Профили диэлектриков
* **C0G**: Линейная характеристика. $C(V)/C_{nom} = 1.0$ при любых напряжениях.
* **X7R**: Умеренная нелинейность. Емкость падает до ~50% при номинальном напряжении ($V = V_{rated}$) и до ~15% при двойном напряжении.
* **X5R**: Сильная нелинейность. Емкость падает до ~20% при номинальном напряжении ($V = V_{rated}$) и до ~5% при двойном напряжении.

Пример нормированного профиля LUT, используемого в решателе:
```python
# Напряжение (V / V_rated)
v_lut = [-2.0, -1.5, -1.0, -0.75, -0.5, -0.25,  0.0, 0.25, 0.5, 0.75, 1.0, 1.5, 2.0]
# Емкость X7R (C / C_nom)
c_x7r = [0.15, 0.25, 0.50,  0.75,  0.90, 0.97,  1.0, 0.97, 0.90, 0.75, 0.50, 0.25, 0.15]
# Емкость X5R (C / C_nom)
c_x5r = [0.05, 0.10, 0.20,  0.40,  0.60, 0.85,  1.0, 0.85, 0.60, 0.40, 0.20, 0.10, 0.05]
```

## 2. Дифференциальные уравнения входного тракта АЦП

Входной тракт состоит из внешнего RC-фильтра ($R_{ext}$, нелинейный $C_{ext}(V_{ext})$) и внутренней цепи выборки-хранения АЦП ($R_{sw}$, линейный $C_{sh}$):

### Схема
```
 V_in(t) ---[ R_ext ]---+---[ Switch (R_sw) ]---+
                        |                       |
                     [C_ext]                 [C_sh]
                        |                       |
                       GND                     GND
```

Система описывается двумя состояниями ключа выборки:

1. **Фаза выборки (Acquisition Phase)**: ключ замкнут ($t \pmod{T_s} < T_{acq}$)
   \[
   \begin{cases}
   \frac{d V_{ext}}{dt} = \frac{1}{C_{ext}(V_{ext})} \left( \frac{V_{in}(t) - V_{ext}}{R_{ext}} - \frac{V_{ext} - V_{sh}}{R_{sw}} \right) \\
   \frac{d V_{sh}}{dt} = \frac{V_{ext} - V_{sh}}{R_{sw} \cdot C_{sh}}
   \end{cases}
   \]

2. **Фаза преобразования (Conversion Phase)**: ключ разомкнут ($t \pmod{T_s} \ge T_{acq}$)
   \[
   \begin{cases}
   \frac{d V_{ext}}{dt} = \frac{V_{in}(t) - V_{ext}}{R_{ext} \cdot C_{ext}(V_{ext})} \\
   \frac{d V_{sh}}{dt} = 0
   \end{cases}
   \]

## 3. Методы решения уравнений

В симуляторе реализовано два метода решения ОДУ:
1. **Radau (`radau`)**: Поцикловое интегрирование с использованием жесткого неявного ОДУ решателя `scipy.integrate.solve_ivp` с автовыбором шага. Обеспечивает максимальную точность, но имеет высокую вычислительную нагрузку в Python.
2. **Полунеявный метод Эйлера (`semi_implicit`)**: Специализированный пошаговый алгоритм, в котором емкость конденсатора берется с предыдущего шага времени ($C_{ext}^{k+1} \approx C_{ext}(V_{ext}^k)$). Это позволяет свести нелинейную систему к быстрому линейному обновлению напряжений на каждом шаге:
   \[
   V_{ext}^{k+1} = \frac{V_{ext}^k + \frac{dt}{C_{ext}^k} \left( G_{ext} V_{in}^{k+1} + G_{sw} V_{sh}^k \right)}{1 + \frac{dt}{C_{ext}^k} \left( G_{ext} + G_{sw} \right)}
   \]
   Метод в **20–30 раз быстрее** схемы Radau и дает погрешность вычислений менее **12 мкВ** при 100 шагах на период дискретизации, что идеально подходит для выполнения многомерных сканирований в реальном времени.
