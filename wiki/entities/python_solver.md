# Математический решатель: Python Solver Core (python_solver.md)

Расчетное ядро симулятора разработано на языке Python 3 и полностью изолировано от веб-интерфейса, что позволяет тестировать и валидировать его локально.

---

## 1. Структура файлов и каталогов

Математическое ядро расположено в следующих файлах:
* **[src/solver.py](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/src/solver.py)** — генерация сигналов, модели нелинейных диэлектриков, интегрирование ОДУ и квантование.
* **[src/analyzer.py](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/src/analyzer.py)** — расчет FFT, THD, SNR, ENOB, ошибки установления и многопараметрический свип.
* **[src/capacitors.json](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/src/capacitors.json)** — единый каталог пресетов конденсаторов в СИ (`c_nom` хранится в фарадах), используемый Python-ядром и веб-интерфейсом.
* **[src/verify.py](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/src/verify.py)** — скрипт автономной проверки физической адекватности модели (строит графики).
* **[tests/test_solver.py](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/tests/test_solver.py)** — модульные тесты для проверки корректности отдельных функций.

## 2. Ключевые классы и их интерфейсы

### Класс `SignalGenerator` (в `solver.py`)
Предназначен для генерации значений входного напряжения $V_{in}(t)$.
* **Инициализация**:
  `SignalGenerator(form: str, amplitude: float, frequency: float, dc_offset: float)`
  * `form`: `'sine'` (синусоида), `'square'` (меандр), `'triangle'` (треугольный сигнал).
* **Вызов**:
  Принимает время `t` (скаляр или массив `numpy`) и возвращает соответствующее напряжение в вольтах.

### Класс `CapacitorModel` (в `solver.py`)
Реализует нелинейную модель конденсатора $C(V)$ на основе интерполяции Look-Up Table (LUT) для C0G, X7R, X5R и Custom.
* **Инициализация**:
  `CapacitorModel(cap_type: str, c_nom: float, v_rated: float = 10.0, custom_drop: Optional[float] = None)`
* **Вызов `get_C(v)`**:
  Возвращает емкость в фарадах для приложенного напряжения `v`. Линейно интерполирует значения LUT и отсекает экстраполяцию за пределами $2 \cdot V_{rated}$, предотвращая получение отрицательной или нулевой емкости.

### Каталог пресетов `CAPACITOR_PRESETS`
Справочник компонентов загружается функцией `load_capacitor_presets()` из общего JSON-файла [src/capacitors.json](file:///g:/Мой%20диск/SNR_Lib/ADC_Model/src/capacitors.json). Это устраняет расхождение между Python-ядром и JavaScript-интерфейсом: добавление или исправление компонента теперь выполняется в одном месте.

### Функция `simulate_adc_input` (в `solver.py`)
Основной симуляционный цикл.
* **Параметры**:
  * `signal_gen`: генератор сигнала.
  * `cap_model`: модель конденсатора.
  * `R_ext`, `R_sw`, `C_sh`, `T_acq`, `f_s`, `N_samples`, `V_ref`.
  * `method`: метод решения ОДУ — `'semi_implicit'` (быстрый полунеявный, рекомендован по умолчанию) или `'radau'` (медленный жесткий решатель Radau из SciPy для верификации).
  * `store_trace`: сохранять ли полные временные массивы для графиков. Для batch/sweep-расчетов используется `False`.
  * `trace_until`: опциональное ограничение времени сохраняемой траектории; выборки `v_samples` и метрики при этом считаются по полному `N_samples`.
* **Возвращает**:
  Словарь с непрерывными траекториями сигналов (время, вход, напряжение на $C_{ext}$, напряжение на $C_{sh}$, состояние ключа) и дискретными отсчетами АЦП.
* **Защита входных параметров**:
  Перед расчетом вызывается `validate_simulation_parameters()`, которая отклоняет неположительные сопротивления, емкости, частоты и напряжения, а также режимы, где $T_{acq} \ge T_s$.

### Оптимизация времени расчета
В интерактивном пути сохранена высокая детализация (`steps_per_cycle=100`), так как снижение шага почти не меняет ENOB, но может искажать `Settling Error`. Для фазы acquisition быстрый решатель использует не менее трех внутренних шагов. При умеренной жесткости применяется точный шаг линейной 2x2 RC-системы с емкостью \(C_{ext}\), зафиксированной в начале шага; для очень жестких нелинейных режимов сохраняется L-стабильный backward Euler.

Это устраняет крупный выброс быстрого sweep в точке \(R_{ext}=2\text{ кОм}\), \(C_{ext}=0.1\text{ нФ}\), \(f_{in}=20\text{ кГц}\): пиковое отличие от Radau снижено примерно со `114 mV` до `2.7 mV` при `25` шагах на период дискретизации.

Дополнительное ускорение достигнуто без ухудшения численного результата:
* скалярные вызовы `SignalGenerator` и `CapacitorModel.get_C()` используют легковесные `math`/ручную интерполяцию вместо NumPy-вызовов на каждом шаге;
* `run_parametric_sweep()` вызывает `simulate_adc_input(..., store_trace=False)`, так как sweep использует только выборки и спектральные метрики;
* оси sweep принимают те же имена параметров, что и веб-интерфейс (`sig_frequency`, `sig_amplitude`, `sig_dc_offset`), поэтому выбранная пользователем ось действительно меняет параметры расчета;
* веб-интерфейс передает `trace_until` для сохранения только видимого участка временной диаграммы, при этом FFT продолжает считаться по полным 1024 отсчетам.

Контрольный локальный замер после оптимизации:
* одиночный расчет `semi_implicit`, 1024 отсчета, 100 шагов/цикл: около 150 мс с полной трассой и около 122 мс с `trace_until`;
* sweep 10x10, 1024 отсчета, 25 шагов/цикл: около 2.65 с.

## 3. Анализ и обработка данных (в `analyzer.py`)

### Функция `calculate_fft_metrics`
* Накладывает окно Блэкмана-Харриса на отсчеты АЦП.
* Убирает взвешенное окном среднее значение для отсечки DC-составляющей без ложной низкочастотной энергии при некогерентной записи.
* Выполняет быстрое преобразование Фурье (`scipy.fft.rfft`).
* Ищет основной гармонический пик.
* Рассчитывает частоты первых 10 гармоник с учетом наложения спектра (aliasing) и интегрирует их мощности в полосе основного лепестка окна ($\pm 5$ спектральных бинов).
* Корректирует шумовую мощность спектра на коэффициент компенсации окна Ханна/Блэкмана-Харриса:
  \[
  Window_{correction} = \frac{CG^2}{NPG} \approx 0.4166
  \]
* Рассчитывает THD, SNR, SINAD, ENOB, а также ошибку установления на основе разницы $V_{ext}(t_{n}) - V_{sh}(t_{n})$ в LSB. Это измеряет именно недоустановление ключа выборки, не смешивая его с задержкой внешнего RC-фильтра.
* Функция `assess_spectral_record()` отмечает записи с числом периодов менее четырех, алиасом в DC, близостью к Найквисту и несинусоидальным тестовым сигналом.

## 4. Локальный запуск и верификация

### Запуск модульных тестов
Модульные тесты проверяют корректность математики и отсутствие регрессий. Запускаются командой из корня проекта:
```bash
python -m unittest discover -s tests
```

### Запуск скрипта верификации
Скрипт проводит три симуляции (C0G, X7R, X5R) при одинаковых параметрах сигнала и сохраняет график во временной и частотной области по пути `wiki/assets/verification_results.png`. Запуск:
```bash
python -m src.verify
```
